## Что делать?
1. Склонируйте себе локально этот проект. Используйте git clone
2. Подготовьте компьютер к работе с этим проектом. Вам понадобится установить [Docker](https://www.docker.com/) и использовать [Laravel Sail](https://laravel.com/docs/8.x/sail#installing-sail-into-existing-applications)
3. Сгенерируйте уникальный ключ приложения ```./vendor/bin/sail artisan key:generate```
4. Запустите проект локально и проверьте работоспособность. Для этого откройте
   в браузере URL проекта и вы должны увидеть "Все работает". В случае с Laravel Sail проект будет
   по адресу http://localhost
5. Сделайте [миграцию базы данных](https://laravel.com/docs/8.x/migrations) командой `./vendor/bin/sail artisan migrate`
6. И [заполните ее тестовыми данными](https://laravel.com/docs/8.x/seeding) командой `./vendor/bin/sail artisan db:seed`
7. Сделайте Code Review Pull Request, который есть к этому проекту. Для этого на странице гитхаба, где вы скачали проект,
перейдите на вкладку Pull Request
5. Pull Request умышленно сделан с максимальным количеством ошибок. Ошибки не в функционале, функционал работает.
Ошибки в некорректном использовании Laravel. Вам надо эти ошибки обнаружить и написать в code review в чем ошибка и как надо.
Список документации, которая поможет вам понят, я укажу ниже
6. По результатам ревью у вас будет представление как правильно написать этот функционал
7. Смело создавайте новый git branch от main, реализуйте функционал как положено и сделайте PR
